Predicting Recipe Difficulty from Text Using Machine Learning

Objective Difficulty Ratings Through NLP and Classification

Ruiyang Wu, Nikki Yuan | SI 670 Applied Machine Learning | University of Michigan | Fall 2025

Recipe difficulty labels are subjective and inconsistent

Current Challenges:

  • Inconsistent criteria across platforms
  • Labels vary (AllRecipes vs. Food.com)
  • Users can't match skill level
  • No objective standard

Our Goal:

Create an objective, data-driven system that predicts difficulty from recipe content alone.

Dataset

RecipeNLG: 30,000 recipes from 2.2M+ corpus

Split: 70% train / 15% validation / 15% test

Features: Title, ingredients, directions, source

Multi-signal heuristics create objective labels

EASY (22%):

≤5 steps, ≤8 ingredients, ≤15 words/step

MEDIUM (69%):

Moderate complexity recipes

HARD (9%):

>12 steps OR >17 ingredients OR advanced techniques

Key Innovation:

Cooking verb count distinguishes verbose descriptions from truly complex recipes.

TF-IDF + numeric features capture patterns

Text Features: 1000 TF-IDF features from instructions (unigrams + bigrams)

Numeric Features: Text length, cooking verb count, ingredient complexity

Critical Design:

Excluded labeling features (num_steps, num_ingredients) to prevent data leakage.

Four models compared

1. Random Forest: 100 trees, handles non-linearity

2. Gradient Boosting: Sequential error correction

3. Logistic Regression: Linear baseline, interpretable

4. Naive Bayes: Probabilistic baseline

Random Forest achieves 86% accuracy

RF & Gradient Boosting: 85%+ accuracy, 0.82 F1-score

Per-Class Performance (Random Forest)

💡 Key Finding:

Models learn from semantic content, not memorized rules. The 86% accuracy proves TF-IDF captures meaningful complexity signals from recipe text.

What Models Learn

Top Features:

  • Cooking verbs & techniques
  • Descriptive language patterns
  • Text length & structure

Learned Patterns:

  • Long, detailed → harder
  • Multiple techniques → harder
  • Simple, direct → easier

Performance Analysis

Easy Class:

Precision: 77%, Recall: 69%, F1: 0.73

Some confusion with medium

Medium Class:

Precision: 87%, Recall: 93%, F1: 0.90

Strongest performance (largest class)

Hard Class:

Precision: 96%, Recall: 72%, F1: 0.83

High precision despite smaller size

ML enables objective recipe difficulty prediction

Main Contributions:

  1. Objective labeling system with multi-signal heuristics
  2. Effective TF-IDF + numeric feature engineering
  3. 86% accuracy with balanced performance
  4. Ready for integration into recipe platforms

Future Extensions

1. User Feedback: Incorporate actual ratings, personalization

2. Advanced NLP: BERT, RoBERTa, attention mechanisms

3. Multi-modal: Images, prep time, equipment

4. Domain-Specific: Cuisine-specific models, skill progression

Methodology Pipeline

1. Parse & clean text data

2. Apply labeling rules

3. Extract TF-IDF + numeric features

4. Train & validate models

5. Evaluate on held-out test set
